# スプレッドシート自動同期 セットアップ手順

ブラウザのタスク管理アプリから、ボタン1つでGoogleスプレッドシートに
タスクを書き込めるようにします。所要時間: 約10分 / 費用: 無料 / LINE不要。

---

## STEP 1. スプレッドシートを用意

1. **https://sheets.google.com/** で空のシートを新規作成
2. 名前を「タスク管理」などに変更
3. URLから **ID** をコピー:
   ```
   https://docs.google.com/spreadsheets/d/【ここがID】/edit
   ```

## STEP 2. GAS（受け口）を作る

1. **https://script.google.com/** →「新しいプロジェクト」
2. 最初の `Code.gs` を全部消し、このフォルダの **`Sync.gs` を丸ごと貼り付け**
3. 先頭の `SPREADSHEET_ID` を、STEP 1のIDに書き換える
4. 保存（Ctrl+S）
5. 関数を **`setup`** にして「▶ 実行」→ 初回は承認（自分のGoogleで許可）
   - 「確認されていません」→「詳細」→「（プロジェクト名）に移動」で進む
   - シートに `tasks` タブと見出しが出ればOK

## STEP 3. デプロイしてURLを取得

1. 右上 **「デプロイ」→「新しいデプロイ」**
2. ⚙️ →「ウェブアプリ」
3. 設定:
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員**  ← 重要
4. 「デプロイ」→ **ウェブアプリのURL**（`.../exec` で終わる）をコピー

## STEP 4. アプリに送信先を登録

1. タスク管理アプリ（index.html）を開く
2. 「📤 書き出し」→ **「⚙ 送信先を設定」**
3. STEP 3 のURLを貼り付けて保存

## STEP 5. 使う

- 「📤 書き出し」→ **「☁️ スプレッドシートに送信」** を押す
- 「✅ 送信しました（n件）」と出たら、スプレッドシートを見る → 反映されています
- 押すたびに **最新の状態に同期**（毎回まるごと置き換えなので重複しません）

---

## LINE追加タスクの一元管理（取り込み）

LINEで追加したタスクを、ブラウザアプリのカンバンに**自動で取り込んで一元管理**できます。

### 仕組み
- LINE＝サッと入れる「受信箱」、ブラウザアプリ＝全部を管理する場所
- アプリを開くたびに、LINEで追加した分を**自動でカンバンに取り込み**（💬LINE バッジが付きます）
- 取り込んだ分はLINE側の受信箱から外れる（重複しない／通知も二重にならない）
- 手動で取り込みたいときは ツールバーの **「🔄 LINE取込」** ボタン

### 使うには（このSync.gsを取り込み対応版に更新）
1. このフォルダの **`Sync.gs` を最新版に貼り替え**（`doGet(e)`/`ackLineTasks`/`getActiveLineTasks` が含まれる版）
2. `SPREADSHEET_ID` は **LINE連携と同じスプレッドシートID**（`line_tasks` タブがある表）にする
3. GASで「デプロイ」→「デプロイを管理」→ ✏️編集 →「新バージョン」→デプロイ（**URLは変わりません**）
4. アプリを開き直すと自動で取り込まれます。手動なら「🔄 LINE取込」

> 取り込み時、通知の取りこぼしを防ぐため**全タスクを自動でシートに同期**します（空のときは上書きしません）。

---

## 補足・注意
- 送信されるのは **画面に表示中のタスク**（フィルタ・タグ絞り込みが反映されます）
- 全件送りたいときは、フィルタを「すべて」にしてから送信
- URLは1台のブラウザに保存されます（別のブラウザでは再設定が必要）
- コード変更後は GAS で「デプロイを管理」→ 編集→「新バージョン」→デプロイで反映

## うまくいかない時
| 症状 | 確認 |
|------|------|
| 送信に失敗する | デプロイの「アクセスできるユーザー」が**全員**か / URLが`/exec`で終わるか |
| 反映されない | `SPREADSHEET_ID`が正しいか / `setup`実行で承認したか |
| 「送信しました」だが件数が出ない | 送信は成功。シートを確認すればOK（環境により確認応答が読めないだけ） |
| LINE取込が効かない | Sync.gsを**取り込み対応版**にして再デプロイしたか / `SPREADSHEET_ID`が`line_tasks`のある表と同じか |
| 取り込んでも消えない | 同じLINEタスクが何度も入る場合は、ブラウザのキャッシュ・別ブラウザでないか確認 |
