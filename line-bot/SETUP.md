# LINE連携 セットアップ手順（①期限リマインド ＋ ②チャット追加）

LINEで「タスク追加」「一覧」ができ、さらに**毎朝、期限が近いタスクを自動通知**します。
通知には **アプリで「スプレッドシートに送信」した分** と **LINEで追加した分** の両方が含まれます。

所要時間: 約20分 / 費用: 無料 / 必要: LINEアカウント・Googleアカウント

> ★前提: 先に作った「スプレッドシート同期」が動いていること（アプリ →「📤 書き出し」→「☁️ スプレッドシートに送信」で表に書き込める状態）。
> このLINE連携は、その**同じスプレッドシート**を使います。

---

## 全体の流れ
1. LINEの「入れ物（チャンネル）」を作り、トークン取得
2. GASに `Code.gs` を貼り、**同じスプレッドシートID**を設定
3. デプロイ → Webhook設定
4. 自分のLINEで友だち追加 → 一度メッセージ送信（通知先の登録）
5. 毎朝の自動通知をオン（トリガー設定）

---

## STEP A. LINE Messaging API チャンネルを作る
1. **https://developers.line.biz/console/** を開き、LINEでログイン
2. 「新規プロバイダー作成」（名前は任意 例: MyTasks）
3. その中で **「Messaging API」チャンネルを作成**
4. 作成したチャンネルを開く
5. **「Messaging API設定」タブ** → 下部 **「チャネルアクセストークン（長期）」** を「発行」してコピー
   → `CHANNEL_ACCESS_TOKEN`
6. 同じタブで設定を確認:
   - **「応答メッセージ」= オフ**
   - **「あいさつメッセージ」= 任意**
   - **「Webhookの利用」= オン**（URLはSTEP Cで設定）

---

## STEP B. GASにコードを貼る
1. **https://script.google.com/** →「新しいプロジェクト」
2. 最初の `Code.gs` を全部消して、このフォルダの **`Code.gs` を丸ごと貼り付け**
3. 冒頭の設定を書き換える:
   ```js
   const CHANNEL_ACCESS_TOKEN = '（STEP Aのトークン）';
   const SPREADSHEET_ID       = '（アプリ送信先と"同じ"スプレッドシートID）';
   const REMIND_HOUR          = 8;   // 通知時刻。変えたい時はここ
   ```
   ※ `SPREADSHEET_ID` は、スプレッドシート同期で使っているものと**必ず同じ**にする。
4. 保存（Ctrl+S）
5. （任意）プロジェクト設定 ⚙ で **タイムゾーンが「東京（日本標準時）」** か確認。違えば変更（通知時刻がずれないように）

---

## STEP C. デプロイして Webhook を設定
1. 右上 **「デプロイ」→「新しいデプロイ」**
2. ⚙ →「ウェブアプリ」
3. 設定:
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員** ← 重要
4. 「デプロイ」→ 初回は承認（自分のGoogleで許可）→ **ウェブアプリURL**（`.../exec`）をコピー
5. LINEコンソール →「Messaging API設定」→ **Webhook URL** に貼って「更新」→ **「Webhookの利用」をオン** →「検証」で `Success`

---

## STEP D. 友だち追加して、通知先を登録
1. 「Messaging API設定」の **QRコード**から、**自分のLINEでBotを友だち追加**
2. トークで何か送ってみる（例: `ヘルプ`）→ 返信が来ればOK
   - この**最初のメッセージで、あなたが通知の宛先として自動登録**されます（毎朝の通知に必要）
3. ためしに:
   - `牛乳を買う 6/30` → 「✅ 追加しました」
   - `一覧` → 一覧表示
   - `通知` → 今の期限リマインドを表示

---

## STEP E. 毎朝の自動通知をオン
1. GASエディタに戻る
2. 上部の関数選択を **`installDailyTrigger`** にして「▶ 実行」
   - これで毎朝 `REMIND_HOUR` 時（既定8時）に自動通知のトリガーが作られます
3. （テスト）関数 **`testReminder`** を実行すると、今すぐ自分に通知が飛びます
   - ※ 先にSTEP Dでメッセージを1回送っておくこと（宛先登録のため）
   - ※ 期限が「超過・今日・明日」のタスクが無いと、何も送られません（正常）

これで完成です 🎉
毎朝、アプリ分＋LINE追加分の期限がまとまってLINEに届きます。

---

## 使い方（LINEトーク）
| 送る言葉 | 動作 |
|----------|------|
| `牛乳を買う` | タスク追加 |
| `牛乳を買う 6/30` | 期限つきで追加（末尾に M/D） |
| `一覧` | 未完了を**進捗状況ごと**に表示（＋定期タスク／LINE受信箱） |
| `完了 2` / `削除 2` | LINE受信箱（番号付き）の操作 |
| `通知` | 期限リマインド（進捗状況ごと＋定期タスク） |
| `ヘルプ` | 使い方 |

## 表示の内容
- **一覧／通知は進捗状況（未着手 ⚪ / 着手中 🔵 / 対応待ち 🟣 / ペンディング 🟡）ごとにグループ表示**し、各タスクに期日も付きます
- **定期タスクも表示**します。通知では `次回予定` が **4日以内**（＝3〜4日前から）になったものを知らせます
- 通知に出る通常タスクは「超過 / 今日 / 明日」のものです（定期タスクだけ早めに通知）

## 双方向の見える化（アプリ⇄LINE）
- **アプリで追加・変更** → 自動でシートに同期され、LINEの `一覧`／毎朝の通知に出る
- **LINEで追加** → アプリを開くと自動でカンバンに取り込まれる（💬LINEバッジ）
- **管理（完了・編集・削除）はアプリ側**で行う。LINEの `完了/削除` はLINE受信箱（まだ取り込まれていない分）に対して使う
- この `一覧` 拡張を使うには **Code.gs を最新版にして再デプロイ**（`listAll`/`getAppTasksAll` を含む版）

## データの置き場所（重要）
- **同じスプレッドシート内で別タブに分離**しています:
  - `tasks` … ブラウザアプリが送信する分（アプリ同期で**毎回上書き**）
  - `line_tasks` … LINEで追加した分（取り込むと done=TRUE で受信箱から外れる）
  - `recurring` … 定期タスク（アプリが同期。LINE側はここを読んで表示・通知）
- リマインドも `一覧` も上記をまとめて読みます。
- 定期タスクの早め通知の日数は Code.gs の `REC_LEAD_DAYS`（既定4）で変更できます。
- アプリの同期は `tasks` タブだけを書き換えるので、LINE受信箱（未取込分）は消えません。

## うまくいかない時
| 症状 | 確認 |
|------|------|
| 返信が来ない | 「応答メッセージ」オフ / 「Webhookの利用」オン / 「検証」Success か |
| 検証が失敗 | デプロイの「アクセス」が**全員** / URLが`/exec`で終わるか |
| 通知が来ない | `installDailyTrigger`を実行したか / 一度LINEから送信して宛先登録したか / 期限該当タスクがあるか |
| 通知時刻がずれる | GASのタイムゾーンが東京か |
| アプリのタスクが通知されない | `SPREADSHEET_ID`がアプリ送信先と同じか / アプリで「送信」して`tasks`に書き込んだか |

## コードを更新したら
GASで「デプロイ」→「デプロイを管理」→ 編集（✏️）→ バージョン「新バージョン」→デプロイ（URLは不変）。
