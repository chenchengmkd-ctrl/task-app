# LINE連携 セットアップ手順（①期限リマインド ＋ ②チャット追加 ＋ ③AIコーチ）

LINEで「タスク追加」「一覧」ができ、さらに**毎朝、期限が近いタスクを自動通知**します。
LINEで追加したタスクは、その場で**直接Supabaseに書き込まれます**（アプリとの取り込み作業は不要）。

所要時間: 約20分 / 費用: 無料 / 必要: LINEアカウント・Googleアカウント

> ★前提: アプリの「Supabase同期」が動いていること（アプリ →「📤 書き出し」→「☁️ 今すぐ同期」でSupabaseに書き込める状態）。
> LINE連携でもこの**同じSupabaseプロジェクト**（`SUPABASE_URL`/`SUPABASE_ANON_KEY`）を使います。Googleスプレッドシートは使いません。

---

## 全体の流れ
1. LINEの「入れ物（チャンネル）」を作り、トークン取得
2. GASに `Code.gs` を貼り、設定値を書き換える
3. デプロイ → Webhook設定
4. 自分のLINEで友だち追加 → 一度メッセージ送信（通知先の登録）
5. 毎朝の自動通知をオン（トリガー設定）
6. （任意）AIコーチ・時刻指定リマインドをオン

---

## STEP A. LINE Messaging API チャンネルを作る
1. **https://developers.line.biz/console/** を開き、LINEでログイン
2. 「新規プロバイダー作成」（名前は任意 例: MyTasks）
3. その中で **「Messaging API」チャンネルを作成**
4. 作成したチャンネルを開く
5. **「Messaging API設定」タブ** → 下部 **「チャネルアクセストークン（長期）」** を「発行」してコピー
   → `CHANNEL_ACCESS_TOKEN`
6. 同じタブで設定を確認:
   - **「応答メッセージ」= オフ**
   - **「あいさつメッセージ」= 任意**
   - **「Webhookの利用」= オン**（URLはSTEP Cで設定）

---

## STEP B. GASにコードを貼る
1. **https://script.google.com/** →「新しいプロジェクト」
2. 最初の `Code.gs` を全部消して、このフォルダの **`Code.gs` を丸ごと貼り付け**
3. 冒頭の設定を書き換える:
   ```js
   const CHANNEL_ACCESS_TOKEN = '（STEP Aのトークン）';
   const SUPABASE_URL         = '（アプリと同じSupabase URL）';
   const SUPABASE_ANON_KEY    = '（アプリと同じSupabase anonキー）';
   const GEMINI_API_KEY       = '（AIコーチ機能を使う場合はGeminiのAPIキー・無料）'; // https://aistudio.google.com/apikey
   const REMIND_HOUR          = 8;   // 通知時刻。変えたい時はここ
   const AGENT_HOUR           = 20;  // AIコーチの進捗チェックイン時刻
   ```
   ※ `SUPABASE_URL` / `SUPABASE_ANON_KEY` は、アプリの `index.html` 内で使っているものと**必ず同じ**にする。
4. 保存（Ctrl+S）
5. （任意）プロジェクト設定 ⚙ で **タイムゾーンが「東京（日本標準時）」** か確認。違えば変更（通知時刻がずれないように）

---

## STEP C. デプロイして Webhook を設定
1. 右上 **「デプロイ」→「新しいデプロイ」**
2. ⚙ →「ウェブアプリ」
3. 設定:
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員** ← 重要
4. 「デプロイ」→ 初回は承認（自分のGoogleで許可）→ **ウェブアプリURL**（`.../exec`）をコピー
5. LINEコンソール →「Messaging API設定」→ **Webhook URL** に貼って「更新」→ **「Webhookの利用」をオン** →「検証」で `Success`

---

## STEP D. 友だち追加して、通知先を登録
1. 「Messaging API設定」の **QRコード**から、**自分のLINEでBotを友だち追加**
2. トークで何か送ってみる（例: `ヘルプ`）→ 返信が来ればOK
   - この**最初のメッセージで、あなたが通知の宛先として自動登録**されます（毎朝の通知に必要）
3. ためしに:
   - `牛乳を買う 6/30` → 「✅ 追加しました」
   - `一覧` → 一覧表示
   - `通知` → 今の期限リマインドを表示

---

## STEP E. 毎朝の自動通知をオン
1. GASエディタに戻る
2. 上部の関数選択を **`installDailyTrigger`** にして「▶ 実行」
   - これで毎朝 `REMIND_HOUR` 時（既定8時）に自動通知のトリガーが作られます
3. （テスト）関数 **`testReminder`** を実行すると、今すぐ自分に通知が飛びます
   - ※ 先にSTEP Dでメッセージを1回送っておくこと（宛先登録のため）
   - ※ 期限が「超過・今日・明日」のタスクが無いと、何も送られません（正常）

これで完成です 🎉
毎朝、期限が近いタスクがまとまってLINEに届きます。

---

## STEP F. AIコーチ（進捗チェックイン・相談）をオン（任意）
AIコーチは **Google Gemini API（無料枠）** を使います。クレジットカード登録は不要です。

1. [Google AI Studio](https://aistudio.google.com/apikey) にGoogleアカウントでログインし、「Create API key」でキーを発行
   - プロジェクト選択を求められたら「Default Gemini Project」を選ぶのがおすすめ
   - 発行された `AIza...` から始まるキーを `GEMINI_API_KEY` に貼り付けて保存
2. GASエディタで関数選択を **`installAgentTrigger`** にして「▶ 実行」
   - 毎晩 `AGENT_HOUR` 時（既定20時）に、AIコーチが進捗をチェックしてLINEに自動送信します
3. （テスト）関数 **`testAgentCheckIn`** を実行すると、今すぐチェックインが届きます
4. トークで `相談` `アドバイス` と送る、または「〜どう？」のような疑問文を送ると、その場でAIコーチが状況を踏まえて返信します
   - ※ 疑問文っぽくない普通の文章（例：`牛乳を買う`）は今まで通りタスク追加として扱われます
5. AIコーチには、タスクの**分解・状態変更・削除**も自然な文章で頼めます:
   - `会議資料の準備を分解して` → サブタスク案が返ってくる。**「はい」と返信すると実際に追加**されます（5分以内に返信しないと提案は流れます）
   - `牛乳を買うを完了にして` → その場でSupabase上のタスクの状態を変更します（同じタイトルのタスクが複数あると変更されず確認メッセージが返ります）
   - `牛乳を買うを削除して` → その場で削除します（同上、複数一致時は確認メッセージ）
   - `このタスクやる意味ある？` のような問いにも、率直な意見を返します

---

## STEP G. 時刻指定タスクの直前リマインドをオン（任意）
時間が決まっているタスク（例：15時から会議）は、開始 `TIME_LEAD_MINUTES` 分前（既定10分前）にも別途LINE通知されます。

1. **Supabaseで列を追加**（タスクに時刻を持たせるため）。Supabaseの SQL Editor で以下を実行:
   ```sql
   ALTER TABLE tasks ADD COLUMN due_time text;
   ```
2. GASエディタで関数選択を **`installTimedReminderTrigger`** にして「▶ 実行」
   - `TIME_CHECK_INTERVAL` 分おき（既定5分おき）にチェックし、開始10分前になったタスクを通知します
3. （テスト）関数 **`testTimedReminders`** を実行すると、今すぐチェックが走ります（対象があれば届きます）
4. 時刻の設定方法:
   - **アプリ側**: タスク追加・編集フォームの期限日の隣にある時刻欄で設定
   - **LINE側**: `会議の準備 6/25 15:00` または `今日17時に〇〇する` のように送る

---

## STEP H. AIコーチに「記憶」を持たせる（任意）
AIコーチが、過去の相談内容や直近完了したタスクの傾向も踏まえて答えられるようにします。

1. **Supabaseでテーブルを作成**。Supabaseの SQL Editor で以下を実行:
   ```sql
   CREATE TABLE ai_log (
     id bigserial primary key,
     user_id text,
     user_text text,
     ai_reply text,
     created_at timestamptz default now()
   );
   ```
2. これだけでOKです。以降、AIコーチとの会話（`相談`／`〜どう？`／分解・状態変更・削除の依頼など）は自動的にこのテーブルに記録され、次回以降の`相談`や毎晩のチェックインで「直近14日で完了したタスク」「直近のやり取り」として参照されます。
3. テーブルを作らなくてもエラーにはなりません（記録に失敗しても、ただ履歴が蓄積されないだけです）。

## 使い方（LINEトーク）
| 送る言葉 | 動作 |
|----------|------|
| `牛乳を買う` | タスク追加（その場でSupabaseに反映） |
| `牛乳を買う 6/30` | 期限つきで追加（末尾に M/D） |
| `会議の準備 6/30 15:00` | 期限＋時刻つきで追加（開始10分前にも通知） |
| `今日17時に小室くんにチラシの件伝える` | 文中の日付・時刻を検出して追加（今日／明日／明後日、HH時MM分／HH時／HH:MM対応） |
| `一覧` | 未完了を**進捗状況ごと**に表示（＋定期タスク） |
| `通知` | 期限リマインド（進捗状況ごと＋定期タスク） |
| `ヘルプ` | 使い方 |
| `相談` / `アドバイス` | AIコーチに進捗評価とアドバイスを聞く |
| `〜どう？` のような疑問文 | AIコーチがタスク状況を踏まえて回答（自動判定） |
| `〇〇を分解して` | AIコーチがサブタスク案を提示（「はい」で追加） |
| `〇〇を完了にして` | AIコーチがタスクの状態を変更（タイトルが一意な場合のみ） |
| `〇〇を削除して` | AIコーチがタスクを削除（タイトルが一意な場合のみ） |
| `〇〇の優先度を上げて／下げて` | AIコーチが優先度を変更（タイトルが一意な場合のみ） |
| `〇〇の期限を7/1にして` | AIコーチが期限を変更（タイトルが一意な場合のみ） |
| `要約して` / `言い換えて` / `まとめて` | AIコーチがタスク状況を簡潔に要約・言い換え |
| `優先順位を整理して` / `並べ替えて` | AIコーチが全タスクの優先度見直し案を提示（「はい」で適用） |
| 長い文章・改行・箇条書きを送る | AIが要点だけの簡潔なタスク名に整理してから追加 |
| 期限を指定せずにタスク追加 | その場で「期限はいつにしますか？」と聞かれる（不要なら「なし」） |

## AIコーチの自動化まわり（新機能）
- **期限未設定タスクへの確認**：期限を指定せずに追加すると、追加直後に期限を聞かれます。答えそびれた分は毎晩のチェックインでまとめて棚卸しされ、「◯◯は6/30、△△は今日」のようにまとめて返信すればAIが振り分けて設定します（不要なら「なし」）。
- **優先順位の自動見直し**：毎晩のチェックインで、期限・停滞日数・状態を踏まえた優先順位の見直し案が届きます。「はい」と返信すると適用され、返信しなければ何も変わりません。「優先順位を整理して」と送れば、その場でも同じ提案を出せます。
- これらの確認待ちは一定時間（6時間）で自動的に流れます。期限切れ後にもう一度聞きたい場合は、同じ操作をやり直してください。

## 表示の内容
- **一覧／通知は進捗状況（未着手 ⚪ / 着手中 🔵 / 対応待ち 🟣 / ペンディング 🟡）ごとにグループ表示**し、各タスクに期日も付きます
- **定期タスクも表示**します。通知では `次回予定` が **4日以内**（＝3〜4日前から）になったものを知らせます
- 通知に出る通常タスクは「超過 / 今日 / 明日」のものです（定期タスクだけ早めに通知）

## 双方向の見える化（アプリ⇄LINE）
- **アプリで追加・変更** → Supabaseに同期され、LINEの `一覧`／毎朝の通知にほぼリアルタイムで出る
- **LINEで追加** → その場でSupabaseに書き込まれるため、アプリで「☁️ 今すぐ同期」すればすぐにカンバンへ反映される（💬LINEバッジ）
- **どちらから操作しても同じデータ**なので、取り込み作業は不要

## データの置き場所（重要）
- **タスク・定期タスクはすべてSupabase**（`tasks`／`recurring`テーブル）に一元化されています。アプリ・LINEどちらから追加・変更しても同じテーブルに反映されます。
- 定期タスクの早め通知の日数は Code.gs の `REC_LEAD_DAYS`（既定4）で変更できます。
- Googleスプレッドシートはこのボットでは使用しません（Apps Scriptの実行環境として使うのみ）。

## うまくいかない時
| 症状 | 確認 |
|------|------|
| 返信が来ない | 「応答メッセージ」オフ / 「Webhookの利用」オン / 「検証」Success か |
| 検証が失敗 | デプロイの「アクセス」が**全員** / URLが`/exec`で終わるか |
| 通知が来ない | `installDailyTrigger`を実行したか / 一度LINEから送信して宛先登録したか / 期限該当タスクがあるか |
| 通知時刻がずれる | GASのタイムゾーンが東京か |
| タスクが反映されない・件数が合わない | `SUPABASE_URL`/`SUPABASE_ANON_KEY`がアプリと同じか / Supabaseダッシュボードで直接確認 |
| Supabaseエラー（401など） | `SUPABASE_ANON_KEY`をSupabaseの「Project Settings → API」から取り直して1行で貼り付け直す |
| AIコーチが応答しない | `GEMINI_API_KEY`を設定したか / [Google AI Studio](https://aistudio.google.com/apikey)でキーが有効か / モデル名（`GEMINI_MODEL`）が使用可能か |
| チェックインが届かない | `installAgentTrigger`を実行したか / 一度LINEから送信して宛先登録したか |
| 時刻指定タスクの直前リマインドが来ない | `installTimedReminderTrigger`を実行したか / Supabaseに`due_time`列を追加したか / タスクに時刻が入っているか |
| 「〇〇を完了にして／削除して」が反映されない | 同じタイトルのタスクが複数ないか（複数あると曖昧なため実行されません） / タイトルが完全一致しているか |
| 分解の提案後「はい」と送っても追加されない | 提案から5分以上経っていないか（期限切れの場合は再度「分解して」から） |

## コードを更新したら
GASで「デプロイ」→「デプロイを管理」→ 編集（✏️）→ バージョン「新バージョン」→デプロイ（URLは不変）。
